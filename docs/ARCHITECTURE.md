## Overview

**OpenClaw** is an Extension Marketplace Platform connected to IzziAPI — a desktop
app where users browse, install, and run AI agent extensions, plus the backend
marketplace API and the developer tooling to build those extensions.

- **Repo type:** pnpm + Turbo monorepo (`apps/*`, `packages/*`, `extensions/*`).
- **Languages (indexed):** TypeScript 74, TSX 33, Python 32, YAML 5, JS 2 (146 files).
- **Stack:**
  - Desktop: **Electron** (main + preload + renderer) + **Vite** + **React** + **Zustand**, local **better-sqlite3**, **Supabase** auth, `electron-updater`.
  - API: **Hono** on `@hono/node-server` + **Supabase** (Postgres), runs on port 8788.
  - Tooling: Node packages for the `.ocx` extension SDK/CLI and `.oab` agent-bundle format.
- **Entry points:** `apps/desktop/src/main/index.ts` (Electron main), `apps/marketplace-api/src/index.ts` (HTTP API), `packages/cli/src/bin/openclaw.ts` (CLI).

## Architectural Layers

### Desktop app — `apps/desktop` (`@openclaw/desktop`)
Electron main process (privileged) + React renderer (UI), bridged by `preload.ts` over IPC.

- **Main / bootstrap** — `main/index.ts`: app lifecycle, `BrowserWindow`, custom `openclaw://` OAuth protocol, wires every service and IPC handler.
- **Auth** — `main/auth/auth-manager.ts`: Supabase client, session persistence via `safeStorage`, demo-mode fallback.
- **Data** — `main/db/`: `database.ts` (DatabaseManager over better-sqlite3), `migrations.ts`, `sqlite-schema.ts`.
- **Agents** — `main/agent/` (agent-service, managed provider, stream-parser) + `main/agents/` (agent-manager, IPC handlers, `gateway-bridge`, `hermes-runtime`).
- **Extensions runtime** — `main/extensions/`: `extension-host` (forked child process per extension), `extension-loader`, `extension-runner`, `ocx-installer`/`ocx-manifest`, `permissions`, `marketplace-download`, `update-checker`.
- **Setup / routing / cost** — `main/setup/`: setup-wizard, `smart-router`/`nine-router-service`, `budget-manager`, `cost-gate`, `model-pricing`, `docker-compose-generator`.
- **Infra / sync / integrations / onboarding / updater** — `main/infra/` (audit-log, device-guard, memory-system), `main/sync/sync-engine`, `main/integrations/`, `main/onboarding/`, `main/updater/`.
- **Renderer (UI)** — `renderer/`: `App.tsx` + `pages/` (Marketplace, AgentStore, Chat, Extensions, CostDashboard, Developer*, Settings, SetupWizard, …), `components/`, `store/` (Zustand: agentWorkspace, agentGateway), `lib/api-client.ts`, i18n (en/vi).

### Marketplace API — `apps/marketplace-api` (`@openclaw/marketplace-api`)
Hono HTTP service backing the marketplace. Forked from the izzi-backend pattern.

- **Bootstrap** — `src/index.ts`: Hono app, CORS, logger, health checks, mounts routes under `/api/*`.
- **Routes** — `routes/`: `extensions`, `uploads`, `reviews`, `developers`, `agents`.
- **Auth middleware** — `middleware/auth.ts`: validates Supabase JWT → `DashboardUser`.
- **Data / libs** — `db/client.ts` (Supabase + demo mode), `db/schema.sql`, `lib/` (storage, sanitize, demo-store), `scripts/seed.ts`.

### Shared packages — `packages/*`
- **`@openclaw/extension-sdk`** — SDK to build `.ocx` extensions (the API extensions compile against).
- **`@openclaw/cli`** — `openclaw` CLI (commander): `init`, `build`, `pack`, `bundle`, `dev` for extension authors.
- **`@openclaw/agent-bundle`** — `.oab` agent-bundle format: `builder`, `validator`, `installer`, `manifest`.

## Key Flows

### 1. Desktop startup
1. `main/index.ts` — Electron `app` ready
2. construct `DatabaseManager` → `AuthManager` (Supabase) → `SyncEngine`
3. construct `ExtensionManager` + `ExtensionLoader`, `AgentService`, `IntegrationsService`, `OnboardingService`, `UpdaterService`, `SetupWizardService`
4. `registerAgentIpcHandlers(...)` + create `BrowserWindow` → load renderer

### 2. Run an installed extension (sandboxed)
1. `ExtensionManager` resolves an installed `.ocx` + granted `permissions`
2. `ExtensionHost.start()` — `fork()` `extension-runner.js` as a child process (IPC channel)
3. host ↔ extension exchange requests; `extension-api.ts` exposes the `ExtensionUIAPI` surface

### 3. Install from marketplace
1. Renderer page (Marketplace/AgentStore) → `lib/api-client.ts` → marketplace-api
2. API `routes/extensions` (+ `uploads`) serves metadata/download; `middleware/auth` checks the Supabase JWT
3. desktop `extensions/marketplace-download.ts` + `ocx-installer.ts` install locally

### 4. Marketplace API request
1. `marketplace-api/src/index.ts` (Hono) → CORS + logger → `app.route('/api/...')`
2. route handler → `getUserFromToken` (auth) → `db/client.ts` (Supabase, or demo store)
